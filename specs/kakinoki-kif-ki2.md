# 柿木将棋の KIF・KI2 フォーマット仕様

柿木将棋 Ⅸ（V9.81）が出力するファイルのフォーマットをサンプルから分析した結果をまとめる。

---

## 共通事項

### 文字コード

拡張子によって文字コードが異なる。

| 拡張子 | 文字コード |
|--------|-----------|
| `.kif` / `.ki2` | **Shift_JIS** |
| `.kifu` / `.ki2u` | **UTF-8** |

### メタデータ区切り文字

フィールド名とその値の区切りには **全角コロン `：`**（U+FF1A）を使用する。

```
開始日時：2024/05/24(金) 01:21:14
手合割：平手
先手：藤井 聡太
```

### 日時フォーマット

2 種類の形式が確認されている。

| 形式 | 例 | 用途 |
|------|-----|------|
| `YYYY/MM/DD(曜) HH:MM:SS` | `2024/05/24(金) 01:21:14` | 対局モード（AI 同士など） |
| `YYYY/MM/DD HH:MM:SS` | `2024/05/24 01:06:17` | 通常（人間が絡む対局等） |

**時刻の区切り**は原則 **半角コロン `:`**。ただし棋譜解析モードでエクスポートしたファイルでは**全角コロン `：`** になるケースあり（例: `01：21：14`）。

### 手合割のパディング

`手合割：` の値は、表示幅が**全角 4 文字**になるよう末尾に全角スペースを補う。

| 手合割 | 実際の値 | 全角文字数 |
|--------|----------|-----------|
| 平手 | `平手　　` | 2 + 2スペース = 4 |
| 角落ち | `角落ち　` | 3 + 1スペース = 4 |
| 二枚落ち | `二枚落ち` | 4 + 0 = 4 |
| 十枚落ち | `十枚落ち` | 4 + 0 = 4 |

### 持駒・盤面（途中局面の場合）

局面が初手からでない場合、プレイヤー情報の前に盤面を記載する。

```
後手の持駒：角　金　香　歩
  ９ ８ ７ ６ ５ ４ ３ ２ １
+---------------------------+
| 龍 ・ ・ ・ ・ ・ ・ ・ ・|一
...
+---------------------------+
先手の持駒：飛　角　金二　銀二　桂　香　歩九
```

- 持駒の駒間は**全角スペース**で区切る
- 枚数は「二」「三」などの漢数字を駒名の直後に付ける（スペースなし）
- 相手の駒は先頭に `v` を付ける（`v歩`、`v玉` など）

### 駒名の表記バリエーション

成飛車（竜王）の表記は `龍` と `竜` の両方が使われる。

### コメント

- 行頭 `*` で始まる行はコメント
- `*` のみの行 = 空コメント行
- `*#` で始まるコメントは柿木将棋の内部情報（形勢値、定跡情報など）

### ブックマーク（図・ラベル）

```
&開始局面
&基本図
&第1図
&変化図
```

行頭に `&` を付け、任意のラベル名を記述する。

### 変化・分岐

```
変化：22手
  22 ８二飛(86)   ( 0:00/00:00:22)
...
```

`変化：{N}手` という行で分岐の起点となる手数を示し、以降に変化手順を記述する。

### 終局表記（最終行）

```
まで105手で先手の勝ち
まで53手で中断
まで72手で先手の入玉勝ち
```

ファイル末尾は空行 1 行で終わる。

## KIF フォーマット固有の仕様

### ファイル先頭行

必ず以下のコメント行で始まる。

```
# --- 柿木将棋Ⅸ V9.81 棋譜ファイル ---
```

KI2 フォーマットにはこの行はない。

### メタデータフィールド一覧

KIF では `手合割：` が常に出力される（平手でも省略しない）。
平手の場合は `先手：` / `後手：`、駒落ちの場合は `下手：` / `上手：` を使用。

確認済みのフィールド:

```
開始日時、終了日時、表題、棋戦、戦型、持ち時間、消費時間、場所、記録係、
掲載、作品番号、作品名、作者、発表誌、発表年月、出典、手数、完全性、分類、
受賞、備考、手合割、先手、後手（または下手、上手）、先手省略名、後手省略名
```

柿木将棋固有フィールド:

```
コンピュータの持ち時間、思考時間制限
```

### 指し手セパレータ行

```
手数----指手---------消費時間--
```

### 指し手行フォーマット

```
   1 ２六歩(27)   ( 0:01/00:00:01)
  10 同　歩(23)   ( 0:01/00:00:05)
 100 ６九龍(99)   ( 0:01/00:00:50)
  17 ２二歩打     ( 0:01/00:00:09)
  41 同　桂(89)   ( 0:01/00:00:21)
```

構造:

```
{手数:4} {指し手テキスト:13表示幅} ( {経過時間}/{累積時間})
```

消費時間フィールドは**省略可能**。省略時は指し手テキストの直後で行が終わる。

#### 手数フィールド（4文字、右詰め）

| 手数 | 表現 |
|------|------|
| 1 | `   1` |
| 10 | `  10` |
| 100 | ` 100` |

#### 指し手テキスト（13表示幅、左詰め）

全角文字は表示幅 2 として計算し、合計 **13 表示幅** になるよう半角スペースで右パディングする。

| 指し手の種類 | 例 | 表示幅 | 末尾スペース |
|------------|-----|-------|------------|
| 通常移動（3全角+移動元） | `２六歩(27)` | 10 | 3スペース |
| 成駒移動（4全角+移動元） | `７八飛成(76)` | 12 | 1スペース |
| 駒打ち（4全角、移動元なし） | `２二歩打` | 8 | 5スペース |
| 同（3全角+移動元） | `同　歩(87)` | 10 | 3スペース |
| 同成（4全角+移動元） | `同　飛成(34)` | 12 | 1スペース |
| 投了・中断（2全角） | `投了` | 4 | 9スペース |
| 入玉勝ち（4全角） | `入玉勝ち` | 8 | 5スペース |

**「同」の表記**: 常に `同　{駒}` の形（同の後に全角スペース 1 文字）。

#### 移動元座標

`(行列)` の形で半角括弧と 2 桁の半角数字。例: `(27)`, `(83)`, `(76)`。

#### 時間フィールド

```
( {経過分:2d}:{経過秒:02d}/{累積時:02d}:{累積分:02d}:{累積秒:02d})
```

- 経過時間: 分は右詰め 2 文字（1桁の場合はスペース）、秒はゼロパディング
- 累積時間: 時・分・秒すべてゼロパディング 2 桁

例:
- `( 0:01/00:00:01)` — 1秒経過、累計1秒
- `( 1:02/00:03:59)` — 1分2秒経過、累計3分59秒

### 特殊終局手

KIF では終局の種別を通常の指し手行と同様に記述する。

```
 106 投了         ( 0:01/00:00:53)
  54 中断         ( 0:13/00:05:11)
  73 入玉勝ち     ( 0:00/00:00:31)
```

### ブランチポイントマーカー

変化の分岐となる手の行末に `+` を付ける場合がある。

```
   9 ５八玉(59)   ( 0:00/00:00:00)+
```

## KI2 フォーマット固有の仕様

### ファイル先頭行

`# ---` のコメント行はない。メタデータから直接始まる。

### メタデータフィールド

KIF との主な違い:

| 項目 | KIF | KI2 |
|------|-----|-----|
| `手合割：` | 常に出力（平手も含む） | 平手は**省略** |
| プレイヤー名 | 空でも `先手：`・`後手：` を出力 | 空の場合は**省略** |

駒落ちの場合: `手合割：` を出力、`下手：` / `上手：` は空なら省略。

### 指し手行フォーマット

指し手の前に手番を示す記号を付ける。

| 手番 | 記号（通常） | 記号（別形） |
|------|------------|------------|
| 先手 | `▲`（U+25B2） | — |
| 後手 | `△`（U+25B3） | `▽`（U+25BD） |

後手の記号は `△` と `▽` の両方が使われる。

時間情報は**記録しない**。

#### 座標数字の全角・半角

指し手の座標数字は、柿木将棋の設定によって**全角**または**半角**から選択できる。半角の場合、算用数字（列）も漢数字（段）もともに半角 ASCII 数字（1–9）になる。

| 設定 | 列（算用数字） | 段（漢数字） | 例 |
|------|-------------|------------|-----|
| 全角（デフォルト） | `２` | `六` | `▲２六歩` |
| 半角 | `2` | `6` | `▲26歩` |

#### 複数指し手の配置

コメントのない指し手は複数手をまとめて 1 行に記述する。1 行あたりの手数は柿木将棋の設定で変更可能（サンプルでは **6 手**）。

```
▲２六歩    △３四歩    ▲７六歩    △３三角    ▲同角成    △同　桂
▲８八銀    △４二玉    ▲２五歩    △２二銀    ▲２四歩    △同　歩
```

コメントが付く手はその手だけを 1 行に置き、コメントを直後に続ける。

```
▲６八金
*#評価値=-180
△１二香
*#評価値=-107
```

#### 各指し手のフィールド幅

各指し手（記号 ▲/△ 込み）は **12 表示幅**を占める。

| 移動テキスト（全角文字数） | 例 | 記号込み表示幅 | 末尾スペース |
|--------------------------|-----|--------------|------------|
| 3全角 | `▲２六歩` | 8 | 4スペース |
| 4全角 | `▲２四歩打` | 10 | 2スペース |
| 5全角 | `▲４六銀上` | 12 | 0スペース |

行末（最終指し手）には末尾スペースを付けない。

#### 「同」の表記

| 場合 | 表記 | 全角文字数 |
|------|------|-----------|
| 1 文字の駒を取る | `同　歩`、`同　飛`、`同　馬` | 3（同 + 全角スペース + 駒1字） |
| 取りながら成る | `同飛成`、`同歩成`、`同角成` | 3（同 + 駒1字 + 成1字） |
| 既成の2文字駒を取る | `同成銀`、`同成桂` | 3（同 + 成1字 + 駒1字） |

ルール: 「同」の後の部分が 1 全角文字のみの場合は**全角スペースを補って** 2 全角文字にそろえる。

#### 不成

```
▲９一銀不成
```

`不成` を末尾に付ける（5全角文字、末尾スペースなし）。

### 終局手の扱い

KI2 では `投了`・`中断`・`入玉勝ち` などの終局手を指し手として記述せず、最終行の結果サマリーのみに反映する。

```
まで105手で先手の勝ち
まで53手で中断
まで72手で先手の入玉勝ち
```

### 分岐の記述

KIF と同様に `変化：{N}手` で分岐を示す。変化内の指し手も 6 手 1 行形式で記述する。

```
変化：22手
△８二飛    ▲３六歩    △４四歩    ▲３七桂    △４二銀    ▲４六歩
△４三銀
```

## 柿木将棋固有コメント（`*#` で始まる行）

### 対局モード（KShogiGame）

```
*#定跡手 [定跡1]:歩26-27
*#[定跡候補]   0:歩26-27  [優先](24392/46348) ...
*#形勢[807] ▲２二歩打  △８七歩打  ...
*#第17手 先手 Lv:10 ランダム (通常) [序盤] 進行度:51 評価値:807 時間:0.02/0.07/0.00 ...
```

### 解析モード（KShogiAnalysis）

```
*#定跡手  ▲２六歩(27)   [優先] 勝率:(24392/46348)
*###[定跡候補] ▲２六歩(27)/優先(24392/46348) ...
*#指し手[-310] 疑問 ▲２二歩打  △同　銀  ...
*#推奨手[157] ▲２四歩打  △４二銀  ...
*#棋譜解析 K-Shogi v.3.9.0(Win) Lv.20 2コア ぴよベンチ:462
```

評価値の形勢判断ラベル（例）:
`疑問`、`悪手`、`好手`、`互角`、`後手有望`、`先手有利`、`先手優勢`、`先手勝勢`

解析モードのファイルは `開始日時` の時刻部分に**全角コロン**（`01：21：14`）を使う場合がある（対局モードは半角コロン）。
