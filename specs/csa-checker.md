# CSA 標準棋譜ファイル形式 検査仕様

参照仕様: CSA 標準棋譜ファイル形式 第７版 V3.0（2024年5月14日）

---

## 行の種別

CSA 棋譜の各行は以下のいずれかに属する。行の種別外の行は「認識できない行」として減点する。

| 行の種別 | 先頭文字・パターン | 説明 |
|---|---|---|
| 文字コード宣言 | `'CSA encoding=` | 先頭行のみ有効 |
| コメント（読み飛ばし） | `'` で始まる | 任意位置に記述可 |
| プログラムが読むコメント | `'*` で始まる | 指し手・局面の後 |
| バージョン | `V` で始まる | `V3.0` など |
| 対局者名 | `N+` / `N-` で始まる | |
| 棋譜情報 | `$` で始まる | キーワード+`:`+データ |
| 局面（一括） | `P1`〜`P9` | 行番号付き局面行 |
| 局面（平手・駒落ち） | `PI` | |
| 局面（駒別） | `P+` / `P-` | |
| 手番 | `+` または `-` のみ1文字 | |
| 通常の指し手 | `+XXXXYYPP` / `-XXXXYYPP` | 7文字 |
| 特殊な指し手・終局 | `%` で始まる | |
| 消費時間 | `T` で始まる | |
| セパレータ | `/` のみ1文字 | 複数棋譜の区切り |
| マルチステートメント | `,` を含む行 | 上記を `,` で連結 |

---

## 検査項目

### A. 文字コード宣言（任意）

- `'CSA encoding=UTF-8` または `'CSA encoding=SHIFT_JIS` であること
  - 値が `UTF-8` / `SHIFT_JIS` 以外であれば違反

### B. バージョン行（任意）

- `V` の後に数字とドットのみ（例: `V3.0`, `V2.2`）
- 既知のバージョン: `V1.0`（無印）, `V2.1`, `V2.2`, `V3.0`
- 不正な文字列が続く場合は違反

### C. 対局者名（任意）

- `N+` / `N-` で始まること
- 名前部分が空でないこと（`N+` のみの行は違反）

### D. 棋譜情報（`$` 行、任意）

- `$KEYWORD:value` の形式であること（`:` が必須）
- 既知のキーワード:
  `EVENT`, `SITE`, `START_TIME`, `END_TIME`, `TIME`, `TIME+`, `TIME-`,
  `TIME_LIMIT`, `OPENING`, `MAX_MOVES`, `JISHOGI`, `NOTE`
  - 未知のキーワードは警告（減点なし）
- `$START_TIME` / `$END_TIME`: 値が `YYYY/MM/DD` または `YYYY/MM/DD HH:MM:SS` の形式
- `$TIME`: 値が `数値+数値+数値` の形式（数値は整数または小数、最大3桁の小数）
- `$TIME_LIMIT`: 値が `HH:MM+SS` の形式
- `$MAX_MOVES`: 値が整数であること
- `$JISHOGI`: 値が `24` または `27` であること

### E. 局面行

#### E-1. 平手・駒落ち（`PI`）

- `PI` の後は `XXPP` の繰り返し（位置2桁 + 駒名2桁）、またはデータなし（平手）
- 駒名は有効な未成駒名であること（FU,KY,KE,GI,KI,KA,HI,OU）
- 位置は `11`〜`99` の範囲

#### E-2. 一括局面行（`P1`〜`P9`）

- `P` + 行番号(1桁) + 9マス分のデータ
- 各マスは3文字: `+PP`, `-PP`, ` * ` のいずれか（`*` は空きマス）
- 合計 `2 + 9×3 = 29` 文字
- 駒名は有効な全駒名（成駒含む）であること

#### E-3. 駒別表現（`P+` / `P-`）

- `P+` / `P-` の後に `XXPP` の繰り返し
- 位置: `11`〜`99`（盤上）または `00`（駒台）
- 駒台（`00`）に `OU`（玉）は置けない
- 駒名 `AL`（残り全部）は駒台（`00`）かつ最後の `P+`/`P-` 行でのみ有効

#### E-4. 手番行

- `+` または `-` の1文字のみ
- 局面行の後、指し手の前に1回だけ現れること

### F. 指し手行

#### F-1. 通常の指し手

- 形式: `[+-][0-9]{4}[A-Z]{2}` の7文字
  - 先後: `+`（先手）または `-`（後手）
  - 移動元: 2桁の数字（`11`〜`99` または `00`）
  - 移動先: 2桁の数字（`11`〜`99`）
  - 駒名: 有効な駒名2文字
- 駒台からの打ち駒: 移動元が `00`、駒名は未成駒（OU以外）

有効な駒名:

| 略称 | 駒 | 成駒 |
|---|---|---|
| FU | 歩 | TO |
| KY | 香 | NY |
| KE | 桂 | NK |
| GI | 銀 | NG |
| KI | 金 | （なし） |
| KA | 角 | UM |
| HI | 飛 | RY |
| OU | 玉 | （なし） |

#### F-2. 特殊な指し手・終局状況（`%` 行）

有効な文字列（V3.0 時点）:

```
%TORYO %CHUDAN %SENNICHITE %TIME_UP %ILLEGAL_MOVE
%+ILLEGAL_ACTION %-ILLEGAL_ACTION %JISHOGI %KACHI %HIKIWAKE
%MAX_MOVES %TSUMI %FUZUMI %ERROR
```

- `%MATTA` は V3.0 で削除済み → 違反として警告
- `%` の後が上記以外の文字列は違反

### G. 消費時間行（`T` 行）

- `T` の後に正の数値（整数または小数、小数点以下最大3桁）
- 例: `T15`, `T6.123`, `T0`
- 指し手行の直後にのみ記述できる（局面行・コメント行の後は違反）
- `T` の後が数値以外は違反

### H. マルチステートメント

- `,` で複数の行を1行に結合できる
- 分割後の各部分を個別に検査する

### I. セパレータ（`/`）

- `/` の1文字のみ
- セパレータにより複数棋譜を区切ることができる

### J. 認識できない行

- 上記のいずれにも該当しない行は減点対象

---

## 順序の制約

仕様では `(6)コメント以外` は以下の順序でなければならない:

```
文字コード宣言（先頭行、省略可）
バージョン（省略可）
棋譜情報（省略可）
開始局面
手番
指し手・消費時間（省略可）
```

順序違反（例：局面行の前に指し手行が来る）は違反として報告する。

## スコアリング方針

| 検査項目 | 重み |
|---|---|
| 認識できない行がない | 3 |
| 指し手行の形式が正しい | 3 |
| 駒名が有効 | 3 |
| `%` 行が有効 | 2 |
| 消費時間行の形式が正しい | 2 |
| 局面行の形式が正しい | 2 |
| 棋譜情報の形式が正しい | 1 |

## tsshogi との差分（実装上の注意）

tsshogi の `importCSA` は寛容なパーサーであり、以下のような非標準入力も受け入れる場合がある:

- 駒名の誤字・略称の誤り
- 手番行の欠落
- 棋譜情報の形式不正

本検査はこれらを独自に検出し、tsshogi がパースできても仕様違反を報告する。
